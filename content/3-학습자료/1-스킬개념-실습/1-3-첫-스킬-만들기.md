
# 1.3 첫 스킬 만들기

> skill-creator 스킬을 사용해서 나만의 첫 스킬을 만들어봅니다.


## skill-creator란?

**스킬을 만드는 스킬**입니다.

| 직접 만들기 | skill-creator 사용 |
|------------|-------------------|
| SKILL.md 문법 공부 필요 | 대화하면서 자동 생성 |
| frontmatter 형식 외워야 함 | 질문에 답하면 완성 |
| 오타나면 에러 발생 | 검증된 구조로 생성 |

> 💡 **비유**: 웹사이트 만들 때 HTML 직접 쓰기 vs 노션/위즈/프리즘 사용하기


## 실습: 첫 스킬 만들기

### Step 0: 실습 폴더 준비

스킬을 테스트하려면 **작업할 폴더**가 필요합니다. 간단한 폴더를 하나 만들어서 VSCode에서 열어볼게요.

**폴더 만들기**

1. **Finder** 열기
2. **Documents**(문서) 폴더로 이동
3. 우클릭 → **새로운 폴더** 만들기
4. 이름: `skill-test` 입력

```
Documents/
└── skill-test/    ← 여기서 실습합니다
```

**VSCode에서 열기**

1. VSCode **왼쪽 상단** 메뉴에서 `파일` 클릭
2. `폴더 열기...` 클릭
3. 방금 만든 `skill-test` 폴더 선택 → `열기`

```
VSCode 왼쪽 상단 메뉴:

파일(F)
├── 새 파일
├── 폴더 열기...   ← 이걸 클릭!
├── ...
```

**Claude 익스텐션 열기**

| Mac | Windows |
|-----|---------|
| `Cmd + Shift + Esc` | `Ctrl + Shift + Esc` |

오른쪽에 채팅창이 나타나면 준비 완료!


### Step 2: 질문에 답하기

**질문 1: "어떤 스킬을 만들고 싶으세요?"**

예시 답변:
```
회의록 요약 스킬을 만들고 싶어요.
긴 회의록을 주면 핵심 결정사항과 액션아이템만 뽑아주는 거예요.
전역 스킬로 만들어줘.
```

> 💡 **"전역 스킬로 만들어줘"** 를 꼭 붙이세요!
> 전역 = 어떤 프로젝트에서든 사용 가능. 안 붙이면 현재 프로젝트에서만 사용됩니다.

**질문 2: "어떤 상황에서 이 스킬을 사용하나요?"**

예시 답변:
```
- 팀 회의 후 회의록 정리할 때
- 길게 적힌 미팅 노트 받았을 때
- "회의 요약해줘", "액션아이템 뽑아줘" 라고 말할 때
```

**질문 3: "결과물은 어떤 형태가 좋을까요?"**

예시 답변:
```
마크다운으로:
- 한 줄 요약
- 핵심 결정사항 (불릿)
- 액션아이템 (담당자, 마감일 포함)
```


### Step 4: 스킬 테스트

**테스트 1**: 스킬 목록에서 확인

Claude에게:
```
사용 가능한 스킬 보여줘
```
→ 방금 만든 스킬이 목록에 보이면 성공!

> 💡 터미널에서는 `/skills` 명령어로도 확인할 수 있어요.

![](../../../public/images/CleanShot%202026-02-22%20at%2004.18.38@2x.png)

**테스트 2**: 테스트용 회의 스크립트 준비

실제 회의 녹음을 텍스트로 변환한 것처럼 만들어둔 예시입니다.
아래 내용을 **복사**하세요:

```markdown
# 프로덕트팀 주간 회의 - 2026.02.20 (목) 14:00

참석자: 김서연(PM), 이준호(디자인), 박민수(개발), 최하늘(마케팅)


**테스트 3**: 스킬로 회의록 요약해보기

이제 만든 스킬을 실제로 테스트합니다:

```
@회의스크립트전문.md 이 회의록 요약해줘
```

→ 스킬이 자동으로 실행되면 성공!


## 💡 잘 만들어진 스킬의 특징

### description이 잘 써졌는가?

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "회의록 요약" | "회의록을 요약합니다. '회의 요약해줘', '액션아이템 뽑아줘'라고 말하면 실행" |
| 한 단어 | **무엇을 하는지 + 언제 쓰는지** 모두 포함 |

> 🎯 **핵심**: description은 스킬의 **트리거 조건**입니다.
> 사용자가 말할 법한 키워드를 꼭 넣으세요!


## ⚠️ 문제 해결

### "skill-creator가 안 돼요"

Claude에게:
```
skill-creator 스킬 설치해줘
```

### "스킬이 자동으로 안 불려요"

1. "사용 가능한 스킬 보여줘"로 설치 확인
2. description에 트리거 키워드 있는지 확인
3. 정확한 키워드 사용 (예: "회의 요약해줘")

### "만든 스킬 위치를 모르겠어요"

- 전역 스킬 위치: `~/.claude/skills/스킬이름/SKILL.md`
- Claude에게 "내 전역 스킬 폴더 위치 알려줘"라고 물어보세요
