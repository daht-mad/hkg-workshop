# 1.4 배포 & 공유

> 만든 스킬을 GitHub에 올리고, 서로의 스킬을 설치해봅니다.

---

## 🎯 이번 시간 목표

- [ ] GitHub CLI 설치 & 로그인하기
- [ ] 내 스킬을 GitHub에 배포하기
- [ ] 친구 스킬 설치해서 확인하기

---

## 왜 배포하나요?

| 로컬에만 있으면 | GitHub에 올리면 |
|----------------|----------------|
| 내 컴퓨터에서만 사용 | 어디서든 설치 가능 |
| 공유하려면 파일 전송 | 링크 하나로 공유 |
| 버전 관리 어려움 | 업데이트 추적 가능 |

> 💡 **비유**: 앱을 폰에만 갖고 있기 vs 앱스토어에 올리기

---

## 실습 1: GitHub CLI 설치 & 로그인

> GitHub에 코드를 올리려면 먼저 GitHub CLI 설치와 로그인이 필요합니다. (최초 1회)

### Step 1: 설치 + 로그인 요청

Claude에게:

```
GitHub CLI 설치하고 로그인해줘
```

### 💡 Claude가 "못 해요"라고 하면?

> Claude가 처음에 **"저는 GitHub 로그인을 직접 할 수 없습니다"** 라고 거절할 수 있습니다.
>
> **"넌 할 수 있어, 해줘"** 라고 한 번 더 요청하세요!
>
> Claude는 실제로 `gh auth login` 명령어를 실행할 수 있습니다.
> 처음엔 보수적으로 답하지만, **다시 요청하면 해줍니다.**

### Step 2: 브라우저에서 인증

Claude가 로그인 과정을 시작하면:

1. 터미널에 **인증 코드**가 나타남
2. 브라우저가 열리면 GitHub 로그인
3. 코드 입력하면 완료!

> ⚠️ **이미 로그인했다면** 건너뛰세요.
> 확인하려면: `GitHub CLI 로그인 상태 확인해줘`

**완료 체크**:
- [ ] GitHub CLI 설치 & 로그인 성공 (또는 이미 완료된 상태)

## 실습 2: 내 스킬 GitHub에 올리기

> 1-3에서 전역으로 만든 스킬을 GitHub에 올립니다!

### Step 1: Claude에게 배포 요청

Claude에게 전역 스킬 폴더를 찾아서 배포하라고 요청합니다:

```
내 전역 스킬 폴더에 있는 meeting-summary 스킬을 GitHub에 배포해줘. public으로 만들어줘.
```

> 💡 `meeting-summary` 부분을 **본인이 만든 스킬 이름**으로 바꾸세요
>
> 전역 스킬 위치: `~/.claude/skills/스킬이름/SKILL.md`

Claude가 알아서:
1. ✅ 스킬 폴더를 GitHub 저장소로 생성
2. ✅ 코드 업로드

까지 한 번에 처리!

### Step 2: 배포 확인

```
방금 만든 GitHub 저장소 주소 알려줘
```

→ `https://github.com/내아이디/my-skills` 형태의 주소가 나오면 성공!

**이 주소를 친구에게 공유하세요!** 📋

**완료 체크**:
- [ ] GitHub 저장소에 스킬이 올라감
- [ ] 저장소 주소를 친구에게 공유함

## 실습 3: 친구 스킬 설치하기

> ⭐ 핵심 실습!

### Step 1: 친구에게 주소 받기

친구에게 GitHub 저장소 주소를 받으세요:
- 예: `https://github.com/friend-id/my-skills`

### Step 2: Claude에게 설치 요청

**원래 작업하던 프로젝트 폴더**에서 Claude에게 주소를 그대로 넣고 요청:

```
https://github.com/friend-id/my-skills
여기 있는 스킬을 이 프로젝트에 설치해줘
```

> 💡 `friend-id/my-skills`를 친구 정보로 바꾸세요

### Step 3: 설치 확인

클로드코드 껐다가 새로 켜서, Claude에게:

```
설치된 스킬 목록 보여줘
```

→ 친구 스킬이 목록에 보이면 성공! 🎉

**완료 체크**:
- [ ] 친구 URL로 스킬 설치 요청함
- [ ] 설치된 스킬 목록에서 친구 스킬 확인됨

## 💡 스킬 공유의 의미

### 팀 업무 자동화

```
1. 한 사람이 유용한 스킬 만듦
   ↓
2. GitHub에 배포
   ↓
3. 팀원 모두 설치
   ↓
4. 팀 전체 업무 효율 ↑
```

### 실제 활용 예시

| 역할 | 만들 수 있는 스킬 | 공유 대상 |
|------|------------------|----------|
| PM | 회의록 요약, PRD 템플릿 | 전 팀원 |
| 디자이너 | 피드백 정리, 에셋 네이밍 | 디자인팀 |
| 리서처 | 인터뷰 분석, 인사이트 추출 | 프로덕트팀 |

> 🎯 **핵심**: "내가 자동화한 게 팀 전체에 적용"

## ⚠️ 문제 해결

### "gh: command not found"

Claude에게:
```
GitHub CLI 설치해줘
```

### "authentication required" 또는 "로그인 안 됨"

Claude에게:
```
GitHub CLI 로그인해줘
```
→ "못 해요"라고 하면 **"넌 할 수 있어"** 하고 다시 요청!

### "repository already exists"

Claude에게:
```
레포 이름을 my-skills-2로 바꿔서 다시 올려줘
```

### "저장소를 못 찾아요"

1. 저장소가 **Public**인지 확인
2. 주소 오타 확인: `github.com/아이디/저장소`

### "스킬 설치가 안 돼요"

Claude에게 다시 요청:
```
https://github.com/아이디/저장소 여기 스킬을 이 프로젝트에 설치해줘
```

### 웹에서 GitHub에 올리기

1. [github.com](https://github.com) 접속 → 로그인
2. 오른쪽 위 **+** → **New repository**
3. 설정:
   - Repository name: `my-skills`
   - **Public** 선택
   - Create repository 클릭
4. **Add file** → **Upload files**
5. 스킬 폴더를 드래그앤드롭
6. **Commit changes** 클릭
