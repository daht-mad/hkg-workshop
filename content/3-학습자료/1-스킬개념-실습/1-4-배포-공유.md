
# 1.4 배포 & 공유

> 만든 스킬을 GitHub에 올리고, 서로의 스킬을 설치해봅니다.


## 왜 배포하나요?

| 로컬에만 있으면 | GitHub에 올리면 |
|----------------|----------------|
| 내 컴퓨터에서만 사용 | 어디서든 설치 가능 |
| 공유하려면 파일 전송 | 링크 하나로 공유 |
| 버전 관리 어려움 | 업데이트 추적 가능 |

> 💡 **비유**: 앱을 폰에만 갖고 있기 vs 앱스토어에 올리기


## 실습 2: 내 스킬 GitHub에 올리기

> 1-3에서 전역으로 만든 스킬을 GitHub에 올립니다!

### Step 1: Claude에게 배포 요청

Claude에게 전역 스킬 폴더를 찾아서 배포하라고 요청합니다:

```
내 전역 스킬 폴더에 있는 meeting-summary 스킬을 GitHub에 배포해줘. public으로 만들어줘.
```

> 💡 `meeting-summary` 부분을 **본인이 만든 스킬 이름**으로 바꾸세요
>
> 전역 스킬 위치: `~/.claude/skills/스킬이름/SKILL.md`

Claude가 알아서:
1. ✅ 스킬 폴더를 GitHub 저장소로 생성
2. ✅ 코드 업로드

까지 한 번에 처리!

### Step 2: 배포 확인

```
방금 만든 GitHub 저장소 주소 알려줘
```

→ `https://github.com/내아이디/my-skills` 형태의 주소가 나오면 성공!

**이 주소를 친구에게 공유하세요!** 📋

**완료 체크**:
- [ ] GitHub 저장소에 스킬이 올라감
- [ ] 저장소 주소를 친구에게 공유함


## 💡 스킬 공유의 의미

### 팀 업무 자동화

```
1. 한 사람이 유용한 스킬 만듦
   ↓
2. GitHub에 배포
   ↓
3. 팀원 모두 설치
   ↓
4. 팀 전체 업무 효율 ↑
```

### 실제 활용 예시

| 역할 | 만들 수 있는 스킬 | 공유 대상 |
|------|------------------|----------|
| PM | 회의록 요약, PRD 템플릿 | 전 팀원 |
| 디자이너 | 피드백 정리, 에셋 네이밍 | 디자인팀 |
| 리서처 | 인터뷰 분석, 인사이트 추출 | 프로덕트팀 |

> 🎯 **핵심**: "내가 자동화한 게 팀 전체에 적용"

<summary>🔧 CLI가 어려우면? (웹에서 직접 업로드)</summary>

### 웹에서 GitHub에 올리기

1. [github.com](https://github.com) 접속 → 로그인
2. 오른쪽 위 **+** → **New repository**
3. 설정:
   - Repository name: `my-skills`
   - **Public** 선택
   - Create repository 클릭
4. **Add file** → **Upload files**
5. 스킬 폴더를 드래그앤드롭
6. **Commit changes** 클릭

</details>
