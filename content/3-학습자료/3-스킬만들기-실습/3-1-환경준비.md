
💡 폴더 이름과 안의 파일은 사람마다 다릅니다. 위는 예시이고, 본인의 스킬 이름으로 된 폴더를 찾으세요.

## 1. 프로젝트 환경 준비

> Claude가 내 폴더를 이해할 수 있도록 작업 환경을 설정합니다.

---

### Step 1: VSCode에서 내 설계서 폴더 열기

⚠️ **주의**: 워크샵 교안 폴더가 아니라, **workshop-prep이 만들어준 설계서 폴더**를 열어야 합니다.

**폴더 여는 방법**

1. VSCode **왼쪽 상단** 메뉴에서 `파일` 클릭
2. `새 폴더 열기...` 클릭
3. 내 설계서 폴더를 찾아서 선택
4. `열기` 클릭

```
VSCode 왼쪽 상단 메뉴:

파일(F)
├── 새 파일
├── 새 폴더 열기...   ← 이걸 클릭!
├── ...
```

**어떤 폴더를 열어야 하나요?**

workshop-prep 스킬이 만들어준 **폴더 자체**를 여세요. `OOO-설계서.md`가 들어있는 그 폴더입니다.

```
✅ 올바른 예
새 폴더 열기 → /Documents/daily-feedback/
  ├── daily-feedback-설계서.md  ← 이 파일이 보여야 합니다
  ├── 사전설문응답.md
  └── 연동가이드/

❌ 잘못된 예
새 폴더 열기 → /Documents/workshop-new/  (워크샵 교안 폴더)
```

---

### Step 2: Claude 익스텐션 열기

VSCode 안에서 Claude 채팅창을 열어야 합니다.

| Mac | Windows |
|-----|---------|
| `Cmd + Shift + Esc` | `Ctrl + Shift + Esc` |

**✅ 확인**: 오른쪽에 채팅창이 나타나면 성공입니다!

---

### Step 3: /init으로 프로젝트 초기화

💡 **왜 하나요?**

`/init`은 Claude에게 "이 폴더가 어떤 프로젝트인지" 알려주는 작업입니다. 마치 새 직원이 들어왔을 때 업무 인수인계 문서를 주는 것처럼, Claude에게 이 폴더의 맥락을 설명하는 `CLAUDE.md` 파일을 만들어줍니다.

**Step 3-1: /init 입력하기**

Claude 채팅창에 아래를 입력하고 Enter를 누릅니다.

```
/init
```

Claude가 폴더를 분석하고 `CLAUDE.md` 파일 초안을 보여줍니다.

**Step 3-2: 승인(Accept) 클릭**

Claude가 파일 생성을 제안하면 `Accept` 버튼을 클릭합니다.

**Step 3-3: 내 스타일 추가하기**

/init이 끝나면, Claude에게 내 상황에 맞는 지침을 추가로 요청합니다. `[본인 직무]`와 `[주로 하는 일]` 부분을 본인에 맞게 수정하세요. Claude가 내 업무 맥락을 더 잘 이해할수록 결과물이 좋아집니다.

```
CLAUDE.md에 아래 내용 추가해줘:
- 항상 한국어로 대화해줘
- 저는 [본인 직무]이고, 주로 [주로 하는 일]을 해. 비개발자 관점으로 설명해줘
```

예를 들어:

```
CLAUDE.md에 아래 내용 추가해줘:
- 항상 한국어로 대화해줘
- 나는 마케터이고, 주로 SNS 콘텐츠 기획과 성과 리포트 작성을 해. 비개발자 관점으로 설명해줘
```

```
CLAUDE.md에 아래 내용 추가해줘:
- 항상 한국어로 대화해줘
- 나는 CS팀 리더이고, 주로 고객 문의 응대와 팀원 피드백 관리를 해. 비개발자 관점으로 설명해줘
```

### 잘 안 되면?

| 증상 | 해결 방법 |
|------|----------|
| Claude 아이콘이 안 보여요 | VSCode Extensions에서 "Claude" 검색 후 설치 확인 |
| /init을 입력했는데 아무 반응이 없어요 | Claude 채팅창이 열려 있는지 확인 후 다시 입력 |
| CLAUDE.md가 안 만들어졌어요 | Accept 버튼을 눌렀는지 확인. 안 눌렀다면 다시 /init |
| 폴더를 잘못 열었어요 | 파일 → 새 폴더 열기로 설계서 폴더 다시 선택 |
