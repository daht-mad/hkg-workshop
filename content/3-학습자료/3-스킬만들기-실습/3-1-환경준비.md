> 내가 직접 설계한 스킬을 Claude와 함께 실제로 만들어봅니다.


### 지금까지 배운 것 (3줄 요약)

- **Part 1**: 스킬이 뭔지 알았고, skill-creator로 나만의 스킬을 만들어 GitHub에 배포했습니다
- **Part 2**: 바이브코딩 개념을 배웠고, CLAUDE.md, /init, MCP, 그리고 Esc Esc / Shift+Tab / @파일 / /compact 같은 필수 기능들을 익혔습니다
- **사전 준비**: `/workshop-prep` 스킬로 내 업무를 분석해 **설계서.md** 를 이미 만들어뒀습니다


### 내 설계서 폴더 확인하기

사전 준비 때 `/workshop-prep` 스킬로 만든 **설계서 폴더**가 필요합니다.

`/workshop-prep`을 실행하면 아래처럼 **폴더가 통째로** 만들어져 있습니다:

```
daily-feedback/                        ← 이 폴더를 VSCode에서 열 거예요
├── 사전설문응답.md                     ← 사전설문 때 입력한 내 답변
├── daily-feedback-설계서.md            ← 스킬 설계 내용 (핵심!)
└── 연동가이드/                         ← 내 스킬에 필요한 서비스별 설정 가이드
    ├── slack.md
    └── google-drive.md
```

💡 폴더 이름과 안의 파일은 사람마다 다릅니다. 위는 예시이고, 본인의 스킬 이름으로 된 폴더를 찾으세요.


## 1. 프로젝트 환경 준비

### Step 2: Claude 익스텐션 열기

VSCode 안에서 Claude 채팅창을 열어야 합니다.

| Mac | Windows |
|-----|---------|
| `Cmd + Shift + Esc` | `Ctrl + Shift + Esc` |

**✅ 확인**: 오른쪽에 채팅창이 나타나면 성공입니다!


### ✅ 완료 체크

- [ ] VSCode에서 내 설계서 폴더가 열려 있다 (왼쪽 파일 목록에 설계서.md가 보임)
- [ ] Claude 채팅창이 열려 있다
- [ ] /init을 실행해서 CLAUDE.md 파일이 생성됐다
- [ ] 내 직무와 언어 설정을 CLAUDE.md에 추가했다

### 잘 안 되면?

| 증상 | 해결 방법 |
|------|----------|
| Claude 아이콘이 안 보여요 | VSCode Extensions에서 "Claude" 검색 후 설치 확인 |
| /init을 입력했는데 아무 반응이 없어요 | Claude 채팅창이 열려 있는지 확인 후 다시 입력 |
| CLAUDE.md가 안 만들어졌어요 | Accept 버튼을 눌렀는지 확인. 안 눌렀다면 다시 /init |
| 폴더를 잘못 열었어요 | 파일 → 새 폴더 열기로 설계서 폴더 다시 선택 |
