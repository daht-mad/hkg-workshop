# 설계서로 스킬 만들기 (2) 계획 세우고 만들기

> 만들기 전에 먼저 계획을 세웁니다. 요리하기 전에 레시피를 확인하는 것처럼요.

---

## 2. 계획 세우기 (Plan 모드)

---

### 왜 계획부터 세우나요?

요리를 시작하기 전에 레시피를 먼저 읽어보는 것을 떠올려보세요. 재료가 뭔지, 순서가 어떻게 되는지 파악하고 시작하면 실수가 줄어듭니다.

스킬 만들기도 마찬가지입니다. Claude에게 바로 "스킬 만들어줘!"라고 하면 Claude가 알아서 하긴 하지만, 중간에 방향이 틀어지면 처음부터 다시 해야 합니다. **Plan 모드에서 계획을 먼저 확인하고 내가 원하는 방향으로 조정한 다음 실행하면**, 한 번에 원하는 결과를 얻을 수 있습니다.

| | 계획 없이 바로 실행 | Plan 모드로 계획 후 실행 |
|---|---|---|
| 속도 | 빠르게 시작 | 계획 시간 필요 |
| 결과 | 방향이 다를 수 있음 | 내가 원하는 방향으로 |
| 수정 | 완성 후 처음부터 | 계획 단계에서 미리 조정 |
| 추천 | 간단한 작업 | **중요한 작업 ✅** |

---

### Step 4: Plan 모드 진입

**Plan 모드란?**

Claude가 "파일을 만들거나 수정하지 않고, 계획만 세워주는 모드"입니다. 내가 `"응, 진행해줘"` 라고 말하기 전까지는 아무것도 건드리지 않습니다.

**Plan 모드 켜는 방법**

채팅 입력창 아래에 모드 선택 버튼이 있습니다. 클릭해서 `Plan` 모드로 전환하세요.

```
💬 채팅 입력창
────────────────────────────────
  [Auto-edit ▾]  ← 이 버튼을 클릭!
────────────────────────────────
         ↓ 클릭하면 목록이 나와요
  ┌──────────────┐
  │ Auto-edit    │
  │ ✅ Plan      │ ← Plan 선택!
  │ Auto-accept  │
  └──────────────┘
```

**✅ 확인 방법**

버튼이 `Plan`으로 바뀌면 성공입니다.

> **터미널로 실행 중이라면?** `Shift + Tab`을 눌러서 Plan 모드로 전환하세요. 토글 방식이라 한 번 더 누르면 해제됩니다.

---

### Step 5: @로 설계서 참조하기

💡 **@란?**

채팅창에서 `@`를 입력하면 내 폴더 안의 파일 목록이 나타납니다. 이 중에서 설계서 파일을 선택하면, Claude가 그 파일 내용을 읽고 대화에 활용합니다. "Claude야, 이 파일 보면서 얘기하자"는 뜻입니다.

**방법**

1. Claude 채팅창 입력창을 클릭
2. `@` 키를 입력
3. 파일 목록이 나타나면 설계서 파일 선택

```
@ 입력 시 나타나는 예시:
┌─────────────────────────┐
│ 📄 설계서.md            │ ← 이걸 선택!
│ 📄 CLAUDE.md            │
│ 📁 .claude/             │
└─────────────────────────┘
```

**⚠️ 파일이 안 보이면?**

VSCode 왼쪽 파일 목록에서 설계서.md 파일을 **우클릭** → **경로 복사** 를 선택해서 직접 경로를 붙여넣으세요.

```
예시:
@/Users/내이름/Documents/내스킬/설계서.md
```

### Step 6: Claude에게 스킬 생성 계획 요청

이제 Plan 모드가 켜져 있고, 설계서도 참조했으니 Claude에게 계획을 요청할 차례입니다.

아래 내용을 **그대로** 입력창에 붙여넣으세요.

```
@설계서.md 를 읽고, 이 설계서를 기반으로 스킬을 만들어줘.
skill-creator 스킬을 활용해줘.
스킬은 이 프로젝트 폴더에 저장해줘.
```

**각 줄이 무슨 뜻인가요?**

| 줄                         | 의미                              |
| ------------------------- | ------------------------------- |
| `@설계서.md 를 읽고,`           | "이 파일을 참고해서"라는 뜻                |
| `이 설계서를 기반으로 스킬을 만들어줘.`   | 설계서 내용대로 스킬을 설계하라는 지시           |
| `skill-creator 스킬을 활용해줘.` | Part 1에서 배운 스킬 생성 도구를 사용하라는 뜻   |
| `스킬은 이 프로젝트 폴더에 저장해줘.`    | 만든 스킬을 현재 폴더에만 저장하라는 뜻 (전역설치 X) |

💡 **전역 vs 프로젝트 복습 (Part 1에서 배운 것)**

| 구분 | 저장 위치 | 사용 범위 |
|------|----------|----------|
| 전역 스킬 | `~/.claude/skills/` | 모든 프로젝트에서 사용 가능 |
| 프로젝트 스킬 | `.claude/skills/` (현재 폴더) | 이 폴더에서만 사용 가능 |

이번 실습에서는 프로젝트 스킬로 먼저 만들고, 마음에 들면 나중에 전역으로 옮길 수 있습니다.

---

### Step 7: 계획 살펴보기 ⭐ 가장 중요한 단계!

Claude가 스킬 생성 계획을 보여줄 것입니다. 실행 전에 반드시 이 계획을 꼼꼼히 확인하세요.

**확인해야 할 4가지**

**✅ 1. 설계서 내용이 반영됐나요?**

내 설계서에 적은 "하는 일", "입력받는 것", "출력하는 것"이 계획에 포함되어 있는지 확인하세요.

**✅ 2. 스킬 이름이 적절한가요?**

스킬 이름은 **영어 소문자 + 하이픈(-)** 으로 만들어야 합니다. 이 이름이 폴더명이자 `/슬래시명령어`가 됩니다.

```
✅ 좋은 스킬 이름                ❌ 아쉬운 스킬 이름
/meeting-summary                /MeetingMinutesSummaryTool
/daily-feedback                 /my-custom-automation-skill-v1
/email-draft                    /회의록정리 (한국어 ❌)
/weekly-report                  /skill1 (의미 없음)
```

💡 **이름 짓기 팁**: "이 스킬이 뭘 하는지" 영어 2~3단어로 표현하세요. 모르겠으면 Claude에게 "이 스킬에 어울리는 영어 이름 추천해줘"라고 물어보세요.

**✅ 3. description에 트리거 키워드가 있나요?**

description은 Claude가 언제 이 스킬을 꺼낼지 판단하는 기준입니다. 내가 실제로 사용할 말투가 포함되어 있어야 합니다.

```
✅ 좋은 예: "회의록을 붙여넣으면 핵심을 요약해주는 스킬.
           '회의록 정리해줘', '이거 요약해줘' 같은 말에 반응함"

❌ 아쉬운 예: "회의 내용을 처리하는 도구"
```

**✅ 4. 단계가 구체적인가요?**

"알아서 해줘"보다 "1. 먼저 이렇게 하고, 2. 그 다음 이렇게 해줘"처럼 구체적일수록 좋습니다.

---

**계획이 마음에 안 들면?**

걱정하지 마세요! 이 단계에서 수정하는 것이 정상입니다.

```
이 부분 수정해줘: 스킬 이름을 "이메일초안"으로 바꿔줘
```

```
이 부분 수정해줘: 출력 형식에 "담당자" 항목도 추가해줘
```

```
이 부분 수정해줘: 트리거 키워드에 "메일 써줘"도 추가해줘
```

여러 번 수정해도 됩니다. 만족스러운 계획이 나올 때까지 대화로 조정하세요.

---

### Step 8: 계획 저장하기 (필수!)

⚠️ **반드시 저장하세요!** 다음 단계에서 대화를 초기화(`/clear`)하기 때문에, 저장하지 않으면 계획이 사라집니다.

```
이 계획을 현재 폴더 하위 plans 폴더를 만들고 거기에 저장해줘.
파일명은 YYMMDD-스킬이름-계획.md 로 해줘.
앞으로도 계획을 세우면 여기에 저장하도록 CLAUDE.md에 추가해줘.
```

Claude가 두 가지를 해줄 겁니다:
1. `plans/260222-meeting-summary-계획.md` 같은 파일 생성 → **Accept 클릭**
2. CLAUDE.md에 "계획은 plans 폴더에 저장" 지침 추가 → **Accept 클릭**

💡 **왜 CLAUDE.md에도 추가하나요?**

CLAUDE.md에 적어두면 앞으로 Claude가 계획을 세울 때 자동으로 plans 폴더에 저장합니다. 한 번 설정해두면 매번 말할 필요 없어요.

---

### Step 9: 대화 초기화 후 계획 실행

여기까지 오면서 대화가 꽤 길어졌습니다. 깨끗한 상태에서 실행하는 게 훨씬 안정적입니다.

**Step 9-1: 대화 초기화**

```
/clear
```

💡 걱정하지 마세요! 계획은 파일로 저장했고, CLAUDE.md도 파일이라 `/clear` 해도 전부 남아있습니다.

**Step 9-2: 계획 파일을 참조해서 실행 요청**

새 대화에서 저장해둔 계획 파일을 `@`로 지정하고 실행을 요청합니다.

```
@plans/YYMMDD-스킬이름-계획.md 이 계획대로 스킬을 만들어줘.
```

예시:
```
@plans/260222-meeting-summary-계획.md 이 계획대로 스킬을 만들어줘.
```

Claude가 파일을 만들거나 수정할 때마다 `Accept` / `Reject` 버튼이 나타납니다. **`Accept`를 클릭**하면 변경이 적용됩니다.

💡 모드를 `Auto-edit` 또는 `Auto-accept`로 바꿔두면 일일이 Accept를 누르지 않아도 자동으로 진행됩니다.

> **터미널로 실행 중이라면?** 영어 선택지가 나올 수 있습니다. **1번을 선택하세요** (새 대화 + 자동 실행).

---

## 3. 스킬 생성 지켜보기

> 계획을 승인했다면, 이제 Claude가 실제로 파일을 만들기 시작합니다. 지켜보세요!

---

### Claude가 지금 하고 있는 것

비유하자면, 설계도(Plan)를 보고 실제 건물을 짓는 단계입니다. 도면이 완성됐으니 이제 시공이 시작됩니다.

**VSCode 좌측 파일 탐색기**를 보면 새 파일이 하나씩 생기는 걸 볼 수 있어요.

```
.claude/
└── skills/
    └── 내-스킬-이름/        ← 이 폴더가 생겨요
        ├── SKILL.md          ← 핵심 파일! 스킬의 두뇌입니다
        ├── scripts/          ← (스킬에 따라) 자동화 스크립트
        └── references/       ← (스킬에 따라) 참고 자료
```

💡 **SKILL.md가 핵심입니다.** 이 파일 하나가 Claude에게 "이 스킬을 어떻게 실행할지" 알려주는 설명서입니다.

---

### Claude가 중간에 물어볼 수 있어요

| Claude가 묻는 것 | 어떻게 답하면 되나요? |
|-----------------|---------------------|
| "이 파일을 수정해도 될까요?" | "응" 또는 **Accept** 클릭 |
| "이 폴더를 만들어도 될까요?" | "응" 또는 **Accept** 클릭 |
| "기존 내용을 덮어써도 될까요?" | 내용 확인 후 "응" 또는 "아니" |

💡 자동 수정 모드(1번 선택)였다면 물어보지 않고 바로 진행합니다.

---

### ✅ 완료 체크

- [ ] Plan 모드로 전환했다
- [ ] @로 설계서.md를 참조했다
- [ ] Claude에게 스킬 생성 계획을 요청했다
- [ ] 계획을 검토하고 4가지 항목을 확인했다
- [ ] 계획을 plans 폴더에 저장했다
- [ ] "응, 진행해줘"로 계획을 승인했다
- [ ] `.claude/skills/` 폴더 안에 내 스킬 이름의 폴더가 생겼다
- [ ] 그 안에 `SKILL.md` 파일이 있다

### 잘 안 되면?

| 증상                    | 해결 방법                            |
| --------------------- | -------------------------------- |
| Plan 모드로 전환이 안 돼요     | 입력창 아래 모드 선택 버튼 클릭 → Plan 선택     |
| @를 입력했는데 파일 목록이 안 나와요 | VSCode에서 올바른 폴더가 열려 있는지 확인       |
| 설계서.md가 @목록에 없어요      | 파일 우클릭 → 경로 복사 후 직접 입력           |
| Claude가 영어로만 대화해요     | "앞으로 한국어로 대화해줘"라고 입력             |
| 계획이 설계서 내용과 너무 달라요    | "다시 설계서를 꼼꼼히 읽고 계획을 다시 세워줘"라고 요청 |
| Claude가 멈춘 것 같아요 | 1-2분 기다려보세요. 파일을 만드는 중일 수 있어요 |
| 파일이 생기다가 오류가 났어요 | "오류가 났어. 다시 시도해줘"라고 입력 |
| 아무 파일도 안 생겨요 | `Esc` 눌러서 중단하고, "계속 진행해줘"라고 입력 |
