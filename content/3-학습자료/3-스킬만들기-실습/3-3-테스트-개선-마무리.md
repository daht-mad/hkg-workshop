# 설계서로 스킬 만들기 (3) 테스트, 개선, 마무리

> 요리를 완성했으면 맛을 봐야죠! 만든 스킬이 제대로 작동하는지 확인하고 개선합니다.

## 4. 스킬 테스트하기

---

### 왜 테스트가 중요한가요?

처음부터 완벽한 스킬은 없습니다. 레시피대로 만들었어도 직접 맛봐야 간이 맞는지 알 수 있는 것처럼, 스킬도 직접 실행해봐야 내가 원하는 결과가 나오는지 알 수 있어요.

> "테스트에서 문제가 나오면 실패가 아닙니다. 그게 바로 테스트의 목적입니다."

---

### 테스트 전 준비: 대화 정리

스킬을 만드는 과정에서 대화가 많이 쌓였습니다. 깨끗한 상태에서 테스트하는 게 좋습니다.

**방법 1: 완전히 새 대화 시작 (추천)**
```
/clear
```

**방법 2: 대화 요약 후 계속**
```
/compact
```

💡 `/clear`를 추천합니다. 스킬은 이미 파일로 저장됐으니, 새 대화를 시작해도 스킬은 사라지지 않아요.

---

### Step 10: 첫 테스트 실행

테스트 방법은 두 가지입니다.

**방법 1: 슬래시 명령어로 직접 실행**
```
/내스킬이름
```

**방법 2: 자연어로 실행 (description에 적어둔 트리거 키워드 사용)**

```
# 회의록 요약 스킬이면:
아래 회의록 요약해줘:

[날짜: 2026-02-22]
[참석자: 김철수, 이영희, 박민준]
오늘 회의에서 다음 달 마케팅 계획을 논의했습니다.
김철수: 소셜미디어 예산을 30% 늘리자고 제안
이영희: 인플루언서 협업도 고려하자고 함
결정사항: 다음 주까지 각자 제안서 작성
```

```
# 고객 피드백 정리 스킬이면:
오늘 받은 피드백 정리해줘:

"배송이 너무 느려요. 3일 걸렸어요."
"상품 품질은 좋은데 포장이 부실했습니다."
"CS 응대가 친절해서 좋았어요."
```

```
# 이메일 초안 스킬이면:
이메일 써줘: 거래처에 납기 연장 요청 (1주일 연장 필요, 이유는 원자재 수급 문제)
```

---

### 💡 테스트 데이터가 없다면?

Claude에게 만들어달라고 요청하세요!

```
이 스킬을 테스트할 샘플 데이터를 만들어줘.
실제 업무에서 쓸 법한 내용으로 만들어줘.
```

---

### 결과 확인하기

| 결과 | 다음 단계 |
|------|----------|
| 기대했던 대로 나왔다! | 🎉 축하합니다! 마무리로 이동 |
| 대충 맞는데 조금 아쉽다 | 다음 섹션 "스킬 개선하기"로 |
| 전혀 엉뚱한 결과가 나왔다 | 다음 섹션 "스킬 개선하기"로 |
| 아예 스킬이 실행 안 됐다 | 아래 트러블슈팅 참고 |

### 잘 안 되면?

**스킬이 실행되지 않아요 (슬래시 명령어가 안 보여요)**

Step 1: Claude에게 확인 요청
```
내 스킬이 설치됐는지 확인해줘
```

Step 2: 없다면 재설치 요청
```
.claude/skills/ 폴더에 있는 스킬을 설치해줘
```

## 5. 스킬 개선하기

> 한 번에 완벽한 스킬은 없습니다. 피드백을 주면 Claude가 고쳐줍니다.

---

### 개선의 기본 원리

스킬 개선은 반복 과정입니다. 여러 번 돌아도 괜찮습니다.

```
테스트 → 부족한 점 발견 → Claude에게 수정 요청 → 다시 테스트
  ↑                                                      ↓
  └──────────────── 만족할 때까지 반복 ──────────────────┘
```

요리로 비유하면: 맛보기 → "좀 짜다" → 물 추가 → 다시 맛보기 → 만족!

---

### 자주 나오는 개선 요청 예시

| 이런 문제가 있나요? | 이렇게 요청하세요 |
|-------------------|-----------------|
| 결과물 형식이 맘에 안 들어요 | "결과물을 표 형태로 바꿔줘" |
| 빠진 항목이 있어요 | "결정사항도 꼭 포함해줘" |
| 너무 길어요 | "요약을 3줄 이내로 짧게 해줘" |
| 너무 짧아요 | "각 항목에 상세 내용도 추가해줘" |
| 자동으로 스킬이 안 불려요 | "description에 '~해줘'라는 키워드 추가해줘" |
| 스킬 이름 바꾸고 싶어요 | "스킬 이름을 OOO으로 바꿔줘" |
| 출력 언어가 영어로 나와요 | "결과물을 항상 한국어로 출력하도록 수정해줘" |
| 특정 형식으로 저장하고 싶어요 | "결과물을 마크다운 파일로 저장해줘" |

---

### 개선 요청할 때 팁

**구체적으로 말할수록 좋아요.**

```
❌ 나쁜 예: "좀 더 잘 만들어줘"

✅ 좋은 예: "회의 요약 결과에서 '결정사항'과 '다음 액션'을 구분해서 표시해줘.
           결정사항은 ✅, 다음 액션은 📌 이모지로 표시해줘."
```

---

### 💡 핵심 습관: 실수를 지침으로 만들기

같은 문제가 반복된다면, SKILL.md에 기록해두면 다음엔 안 생깁니다.

```
이 문제가 다시 안 일어나게 SKILL.md에 주의사항으로 추가해줘
```

---

### ✅ 완료 체크

- [ ] 테스트를 최소 2번 이상 했다
- [ ] 결과물 형식이 내가 원하는 대로 나온다
- [ ] 반복되는 문제를 SKILL.md에 지침으로 추가했다

## 6. 외부 서비스 연동하기 (MCP)

> 내 스킬이 Google Sheets, Gmail, Slack 같은 외부 서비스를 사용해야 한다면?

---

### 먼저 확인: 내 스킬에 연동이 필요한가요?

모든 스킬이 외부 서비스가 필요한 건 아닙니다.

| 이런 스킬이라면 연동 불필요 | 이런 스킬이라면 연동 필요 |
|--------------------------|-------------------------|
| 텍스트 붙여넣고 요약하기 | Google Sheets에서 데이터 가져오기 |
| 파일 내용 분석하기 | Gmail로 이메일 자동 발송 |
| 마크다운 문서 생성하기 | Slack 채널에 메시지 보내기 |
| 보고서 초안 작성하기 | Google Calendar에 일정 추가하기 |

**→ 연동이 불필요한 스킬이라면 이 섹션을 건너뛰고 바로 "마무리"로 이동하세요!**

---

### 연동이 필요하다면?

MCP(Model Context Protocol)는 Claude와 외부 서비스 사이의 "통역사"입니다. 한 번 설치해두면 Claude가 그 서비스를 직접 다룰 수 있게 됩니다.

#### Step 11: 어떤 MCP가 필요한지 확인하기

```
이 스킬을 실행하려면 어떤 MCP가 필요해?
```

Claude가 필요한 MCP 목록을 알려줄 겁니다.

---

#### Step 12: 설정가이드 참고해서 MCP 설치하기

이 워크샵 교안의 `5-설정가이드` 폴더에 서비스별 가이드가 있습니다.

| 연동하고 싶은 서비스 | 참고할 가이드 |
|--------------------|-------------|
| Google Calendar | `5-설정가이드/google-calendar-설정가이드.md` |
| Google Drive | `5-설정가이드/google-drive-설정가이드.md` |
| Google Sheets | `5-설정가이드/google-sheets-설정가이드.md` |
| Slack | `5-설정가이드/slack-설정가이드.md` |
| YouTube | `5-설정가이드/youtube-설정가이드.md` |
| Zoom | `5-설정가이드/zoom-설정가이드.md` |

설정가이드를 열어두고, Claude에게 설치를 요청하세요:

```
Google Sheets MCP 설치해줘
```

---

#### Step 13: 연동 후 스킬 재테스트

```
방금 MCP 설치했어. 스킬 다시 테스트해보자.
[테스트 데이터 또는 요청]
```

---

### 💡 연동 작업 순서 (추천)

```
1단계: 스킬의 핵심 로직이 작동하는지 확인 (텍스트 입출력만으로)
   ↓
2단계: 외부 서비스 연동 추가 (MCP 설치)
   ↓
3단계: 전체 흐름 통합 테스트
```

핵심 로직이 먼저 작동해야, 연동 문제인지 로직 문제인지 구분할 수 있습니다.

---

### ⚠️ 연동 설정이 복잡하다면?

걱정하지 마세요. 완전히 괜찮습니다.

- 워크샵 시간 내에 MCP 연동까지 완료하지 못해도 됩니다
- 스킬 자체가 만들어진 것만으로도 충분히 훌륭합니다
- 연동은 워크샵 이후에 차근차근 설정가이드 보면서 해도 됩니다
- 지금 당장 막힌다면 **강사에게 손들고 도움 요청하세요!**

## 부록 B: 문제 해결 가이드

> 막히면 여기서 먼저 찾아보세요.

---

### "Plan 모드가 안 돼요"

**증상**: 모드 전환이 안 돼요

**해결**:
1. 입력창 아래 모드 선택 버튼 클릭 → `Plan` 선택
2. 버튼이 안 보이면: Claude에게 "계획만 세우고 실행하지 마. 내 승인 받아줘"라고 직접 입력
3. 터미널 사용자: `Shift+Tab`으로 전환 (입력창 포커스 확인 후 시도)

---

### "skill-creator가 없어요"

**증상**: `/skill-creator` 입력해도 명령어를 못 찾아요

**해결**:
```
skill-creator 스킬 설치해줘
```

---

### "스킬이 작동 안 해요"

**증상**: 슬래시 명령어를 입력해도 반응이 없어요

**해결**:
1. 스킬 설치 확인: `현재 설치된 스킬 목록 보여줘`
2. 목록에 없다면: `.claude/skills/ 폴더에 있는 스킬을 다시 설치해줘`
3. description에 트리거 키워드 확인: `SKILL.md의 description 보여줘`

---

### "@로 파일이 안 보여요"

**증상**: @ 입력해도 파일 목록이 안 나와요

**해결**:
1. VSCode 탐색기에서 파일 우클릭 → "상대 경로 복사"
2. 복사한 경로를 `@` 뒤에 붙여넣기

---

### "영어가 나와서 모르겠어요"

**증상**: 선택지나 메시지가 영어로 나와요

**해결**:
- 숫자 선택지가 나오면: `1` 누르고 Enter (보통 1번이 "진행")
- "앞으로 모든 답변을 한국어로 해줘"라고 한 번 말해두기

---

### "Claude가 멈춘 것 같아요"

**증상**: 로딩 표시가 계속 돌아가요

**해결**:
1. 1-2분 기다려보세요 (작업 중일 수 있어요)
