# 2.5 Claude Code 필수 기능 6가지

> 알아두면 작업 효율 3배! 바이브코딩의 핵심 기능 6가지를 빠르게 익힙니다.

---

## 1. 🔐 Permission Modes

> **"Claude의 자율성을 조절하세요."**

### 세 가지 모드

채팅창 하단의 모드 버튼을 **클릭**하면 모드가 바뀝니다.
터미널에서는 `Shift + Tab`으로도 전환할 수 있어요.

![](../../../public/images/CleanShot%202026-02-22%20at%2003.49.05@2x.png)

| 모드 | 설명 | 언제 쓰나요? |
|------|------|-------------|
| **Ask before edit** (기본) | 모든 작업에 허락 받음 | 처음 배울 때, 중요한 파일 다룰 때 |
| **Edit automatically** | 파일 수정 자동 허용 | 익숙해진 후, 빠른 작업 원할 때 |
| **Plan mode** | 계획만 세우고 실행 안 함 | 큰 작업 전 미리보기 |

```
Ask before edit:     "이 파일 수정해도 될까요?" → [허용] 클릭
Edit automatically:  바로 수정 (물어보지 않음)
Plan mode:           "이렇게 할 거예요" (실행은 안 함)
```

### 실습: 모드 확인하기

1. Claude 패널 하단 확인 → 현재 모드가 보입니다
2. 모드 버튼을 **클릭**해서 바꿔보기 (터미널: `Shift + Tab`)
3. 다시 **Ask before edit**로 돌아오기

> 💡 **추천**: 오늘 워크샵에서는 **Ask before edit** 유지하세요!
> 익숙해지면 Edit automatically로 바꾸면 됩니다.

### 💡 Plan 모드 + ultrathink 조합

Plan 모드에서 프롬프트 끝에 **`ultrathink`**를 붙이면 Claude가 **최대한 깊이 사고**합니다.

```
// Plan 모드에서 입력 예시
이 프로젝트를 리팩토링할 계획 세워줘 ultrathink
```

- **Plan 모드** = 넓게 탐색 → "어떤 파일을, 어떤 순서로?"
- **ultrathink** = 깊게 사고 → "왜 이 방법이 최선일까?"

> ⚠️ **참고**: ultrathink는 토큰을 더 많이 소모합니다.
> 오늘 워크샵에서는 사용하지 않지만, 실무에서 복잡한 계획을 세울 때 활용해보세요!

## 2. 📎 @ 파일 참조

> **"이 파일 봐!" 라고 직접 알려주세요.**

### 사용법

채팅창에 `@`를 입력하면 **파일 목록**이 나타납니다.

```
@CLAUDE.md 이 파일 내용 요약해줘
@설계서.md 기반으로 코드 짜줘
@에러로그.txt 이 에러 해결해줘
```

### 언제 쓰나요?

| 상황 | 예시 |
|------|------|
| 특정 파일을 Claude에게 보여줄 때 | `@보고서.md 이거 분석해줘` |
| 여러 파일을 참조할 때 | `@A.md 와 @B.md 비교해줘` |
| 파일 내용 기반으로 작업할 때 | `@설계서.md 대로 구현해줘` |

### 실습: 파일 참조해보기

채팅창에 입력:
```
@CLAUDE.md 이 파일 내용을 한 줄로 요약해줘
```

> 💡 **팁**: Claude가 "무슨 파일이요?" 하고 물을 때
> `@파일명`으로 정확히 알려주면 확실합니다.

## 3. 📸 이미지 붙여넣기

> **"에러 화면? 캡처해서 붙여넣기만 하세요."**

### 사용법

채팅창에 **이미지를 드래그하거나 붙여넣기**(Ctrl/Cmd + V) 하면 Claude가 이미지를 봅니다.

```
1. 에러 화면 스크린샷 캡처 (Cmd + Shift + 4)
2. Claude 채팅창에 붙여넣기 (Cmd + V)
3. "이 에러 해결해줘" 입력
```

### 언제 쓰나요?

| 상황 | 예시 |
|------|------|
| 에러 메시지가 떴을 때 | 스크린샷 + "이거 해결해줘" |
| 디자인 참고할 때 | 이미지 + "이거랑 비슷하게 만들어줘" |
| 글로 설명하기 어려울 때 | 스크린샷 + "여기 이 부분 수정해줘" |

> 💡 **팁**: 비개발자에게 가장 유용한 기능!
> 에러 메시지를 복사할 필요 없이, **보이는 그대로 캡처해서 보여주면** Claude가 알아서 분석합니다.

## 4. 🧹 /compact — 대화 요약해서 토큰 절약

> **"작업 끝날 때마다 /compact. 이게 가장 중요한 습관입니다."**

### 이게 뭔가요?

Claude와 대화하면 **토큰**(대화 용량)이 계속 쌓입니다.
`/compact`를 입력하면 지금까지의 대화를 **요약해서 압축** → 핵심만 남기고 토큰을 확보합니다.

```
/compact                              # 기본 압축
/compact 코드 변경사항 위주로 기억해줘    # 특정 내용 강조해서 압축
```

> 💡 **VS Code 확장 팁**: 채팅창 하단의 **차트 아이콘(📊)**을 보면 토큰이 얼마나 찼는지 한눈에 확인할 수 있어요.
> 이 아이콘을 **클릭**하면 바로 compact가 실행됩니다!

### 왜 미리미리 해야 하나요?

> ⚠️ **토큰 90% 찼을 때 compact하면 늦습니다!**

| | 미리미리 compact | 90% 차서 compact |
|---|---|---|
| **요약 품질** | 방금 한 작업만 요약 → 핵심 정확 | 대화 전체를 한번에 요약 → 중요한 내용 누락 |
| **토큰 소모** | 적은 양만 요약 → 가볍게 처리 | 거대한 대화 요약 → 그것만으로 토큰 폭탄 |
| **작업 연속성** | 맥락 유지된 채 다음 작업 | 맥락 끊기고 "아까 뭐했지?" 발생 |

### 핵심 습관: 작업 끝날 때마다 /compact

```
작업 1 완료 → /compact → 작업 2 시작 → 작업 2 완료 → /compact → 작업 3 시작
```

**비유**: 책상 위에 서류를 쌓아두면? 나중에 한꺼번에 정리하면 뭐가 뭔지 모름.
**하나 끝날 때마다 파일링**하면 항상 깔끔한 상태에서 다음 작업 시작!

![](../../../public/images/CleanShot%202026-02-22%20at%2003.56.03@2x.png)

### 실습: /compact 체험

**Step 1**: 지금까지 대화가 좀 쌓였죠? 채팅창에 입력:
```
/compact
```

**Step 2**: 대화가 요약된 걸 확인
- 이전 대화가 짧은 요약으로 압축됨
- 하지만 맥락(CLAUDE.md, 프로젝트 정보)은 유지!

**완료 체크**:
- [ ] `/compact` 실행 성공
- [ ] 대화가 요약되었지만 맥락은 유지되는 것 확인

## 5. 🗑️ /clear — 대화 완전히 새로 시작

> **"완전히 다른 주제로 넘어갈 때는 /clear."**

### /compact vs /clear

| | `/compact` | `/clear` |
|---|----------|----------|
| **하는 일** | 대화 **요약해서 압축** | 대화 **완전 삭제** |
| **맥락** | ✅ 요약본 유지 | ❌ 다 사라짐 |
| **언제 쓰나** | 같은 작업 이어갈 때 | 완전히 다른 주제 시작할 때 |

```
/compact = 노트 요약해서 정리하기 📝 (핵심만 남김)
/clear   = 노트 찢어서 버리기 📄🗑️ (백지 상태)
```

> 🎯 **정리**:
> - 대화 이어갈 거면 → `/compact`
> - 완전히 새로 시작할 거면 → `/clear`

## 6. ⏪ Checkpoints & Rewind

> **"실수했어도 괜찮아요. 되돌리면 됩니다."**

### 이게 뭔가요?

Claude가 파일을 수정할 때마다 **자동으로 체크포인트**가 생깁니다.
게임의 세이브 포인트처럼, 언제든 이전 상태로 되돌릴 수 있어요.

```
작업 1 → 💾 체크포인트 1
작업 2 → 💾 체크포인트 2
작업 3 → 💾 체크포인트 3 ← 지금 여기
          ↓
      "어, 이거 아닌데?" → 되돌리기 → 체크포인트 2로 복원!
```

### 되돌리는 방법

Claude의 응답 옆에 있는 **되돌리기 버튼(↩️)**을 클릭하세요.
터미널에서는 `Esc` 키를 **빠르게 두 번** 누르면 됩니다.

![](../../../public/images/CleanShot%202026-02-22%20at%2003.55.18@2x.png)

| 상황 | 해결 |
|------|------|
| Claude가 파일을 이상하게 수정했을 때 | 되돌리기 버튼 클릭 (터미널: `Esc` × 2) |
| "아까 상태가 더 좋았는데..." | 되돌리기 버튼 클릭 (터미널: `Esc` × 2) |
| 실험해보다가 원래대로 돌리고 싶을 때 | 되돌리기 버튼 클릭 (터미널: `Esc` × 2) |

### 실습: 되돌리기 체험

**Step 1**: Claude에게 아무 파일이나 수정 요청
```
CLAUDE.md에 "테스트 문장입니다" 추가해줘
```

**Step 2**: 수정된 걸 확인 후, Claude 응답 옆의 **되돌리기 버튼(↩️)** 클릭 (터미널: `Esc` 두 번)

**Step 3**: 파일이 원래대로 돌아왔는지 확인

> 💡 **팁**: 비개발자에게 이 기능은 **생명줄**입니다!
> 실수가 두렵지 않아요. 언제든 되돌릴 수 있으니까.

**완료 체크**:
- [ ] 되돌리기 버튼으로 되돌리기 성공

## 필수 기능 정리

| 기능 | 방법 | 핵심 |
|------|------|------|
| **모드 전환** | 하단 모드 버튼 클릭 | Ask/Edit automatically/Plan 선택 |
| **파일 참조** | `@파일명` | 특정 파일 지정 |
| **이미지 붙여넣기** | Cmd+V / 드래그 | 에러 화면 캡처해서 보여주기 |
| **대화 정리** | `/compact` | 작업 끝날 때마다 요약 |
| **새로 시작** | `/clear` | 완전히 다른 주제로 넘어갈 때 |
| **되돌리기** | 되돌리기 버튼 ↩️ | 실수해도 복원 가능 |

> 🎯 **오늘 꼭 기억할 것**: 되돌리기 버튼으로 언제든 복원!
> 나머지는 쓰다 보면 자연스럽게 익혀집니다.
