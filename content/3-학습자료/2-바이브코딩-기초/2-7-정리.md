# 2.7 정리 및 다음 단계

## 이 섹션에서 배운 것

이 섹션에서 우리는 **Claude와의 협업 방식** 3가지를 배웠습니다:

1. **CLAUDE.md** — Claude의 장기기억 (프로젝트 규칙 저장)
2. **MCP** — Claude의 손발 (외부 서비스 연결)
3. **필수 기능** — 효율적인 협업 기술 (Esc, Shift+Tab, @, /)

→ 이 3가지가 모여야 **"바이브코딩"**이 완성됩니다.

---

## 한눈에 정리

| 개념 | 한 줄 요약 |
|------|-----------|
| CLAUDE.md | Claude의 뇌 - 프로젝트 맥락을 기억하는 곳 |
| /init | CLAUDE.md 자동 생성 (언제든 다시 실행 가능) |
| MCP | 외부 서비스 연결 (AI용 USB-C) |
| **Esc Esc** | 되돌리기 - 실수해도 복원 가능 ⭐ |
| **Shift+Tab** | 모드 전환 - Ask/Edit automatically/Plan |
| **@파일명** | 파일 참조 - 특정 파일 지정 |
| **/명령어** | 슬래시 명령 - /compact, /help 등 |

> 🎯 **핵심 습관**:
> - CLAUDE.md는 **직접 수정하지 말고** Claude에게 요청
> - 에러 해결 후 → "같은 실수 안 하게 지침 추가해줘"
> - 실수했으면 → `Esc Esc`로 되돌리기
> - 대화 너무 길어지면 → `/compact`로 정리
