# 2.2 CLAUDE.md란?

> Claude의 "뇌"를 이해합니다.

---

## 🧠 Claude의 "뇌"라고 생각하세요

**CLAUDE.md = 이 프로젝트에서 Claude가 기억하는 모든 것**

사람도 새 프로젝트 들어가면 온보딩 문서 받죠?
- "우리 팀은 이렇게 일해요"
- "이 프로젝트는 이런 거예요"
- "이건 주의해야 해요"

Claude도 똑같습니다. **CLAUDE.md에 적어두면 Claude가 매번 기억합니다.**

---

## 언제 읽히나요?

Claude가 대화를 시작할 때 **자동으로** 읽습니다.

```
폴더 열기 → Claude 대화 시작 → CLAUDE.md 자동으로 읽음 → 맥락 파악 완료
```

그래서 매번 "한국어로 해줘", "나는 PM이야" 같은 말을 반복할 필요가 없어요.

---

## 어떤 내용이 들어가나요?

| 카테고리 | 예시 |
|---------|------|
| **프로젝트 설명** | "이 프로젝트는 고객 피드백을 분석하는 도구야" |
| **작업 스타일** | "한국어로 대화해줘", "코드보다 기획 관점으로 설명해줘" |
| **주의사항** | "파일 삭제 전에 꼭 확인받아" |
| **주요 파일 위치** | "설계서는 docs/설계서.md에 있어" |
| **과거 실수 기록** | "저번에 A 파일 잘못 삭제했으니 조심해" |

---

## 실제 예시

```markdown
# 프로젝트: 고객 피드백 분석 스킬

## 개요
사전설문 기반으로 설계된 피드백 정리 자동화 스킬

## 주요 파일
- daily-feedback-설계서.md: 스킬 설계 문서
- 사전설문응답.md: 사전 설문 응답 원본

## 작업 스타일
- 한국어로 대화
- 비개발자 관점으로 설명
- 이모지 사용 OK

## 주의사항
- 원본 파일은 절대 수정하지 마
- 분석 결과는 항상 /output 폴더에 저장
```

**핵심**: 내용은 프로젝트마다 다릅니다. 폴더 안의 파일에 따라 **맞춤형**으로 작성됩니다.

---

## 어떻게 만드나요? → /init

CLAUDE.md를 직접 작성할 필요 없습니다. `/init` 명령어를 입력하면 Claude가 **자동으로** 만들어줍니다.

### /init이 하는 일

```
1. 폴더 스캔          → 현재 폴더의 파일들을 읽음
      ↓
2. 패턴 파악          → 어떤 프로젝트인지, 무슨 기술 쓰는지 분석
      ↓
3. CLAUDE.md 작성     → 분석 결과를 파일로 정리
      ↓
4. 매 대화마다 참조    → 이후 모든 대화에서 이 파일을 자동으로 읽음
```

### Claude가 자동으로 분석하는 것들

| 분석 대상 | 예시 |
|----------|------|
| 프로젝트 구조 | 어떤 폴더와 파일이 있는지 |
| 기술 스택 | React, Python, Node.js 등 |
| 설정 파일 | package.json, 기존 설정 파일 등 |
| 실행 명령어 | 빌드, 테스트, 실행 방법 |
| 기존 문서 | README.md, 설계서 등 |

> 💡 **비개발자라면?** Claude가 여러분의 설계서, 설문 응답 등 **문서 파일**을 분석해서
> "이 프로젝트는 피드백 정리를 자동화하려는 거구나" 같은 맥락을 파악합니다.

### /init 사용 흐름

```
채팅창에 /init 입력
      ↓
Claude가 폴더 분석 (몇 초 소요)
      ↓
CLAUDE.md 초안 제안
      ↓
"적용(Accept)" 클릭
      ↓
완성! 이후 대화에서 자동 참조됨
```

> 💡 실제로 `/init`을 실행해보는 실습은 **Part 3 (스킬 만들기)** 에서 합니다.

---

## CLAUDE.md가 저장되는 위치

CLAUDE.md는 **여러 곳**에 둘 수 있고, Claude는 모두 읽습니다.

| 위치 | 범위 | 용도 |
|------|------|------|
| **프로젝트 폴더** (`./CLAUDE.md`) | 이 프로젝트에서만 | 프로젝트 규칙, 구조 설명 |
| **홈 폴더** (`~/.claude/CLAUDE.md`) | 내 모든 프로젝트 | 개인 선호 (한국어, 스타일 등) |

```
~/.claude/CLAUDE.md          ← 모든 프로젝트에 적용 (개인 설정)
      +
./CLAUDE.md                  ← 현재 프로젝트에만 적용
      =
Claude가 둘 다 읽고 합쳐서 이해
```

> 💡 **비유**: 회사 공통 업무 규칙 + 우리 팀만의 규칙 = 합쳐서 적용

---

## 💡 CLAUDE.md 잘 쓰는 습관

### 습관 1: 직접 수정하지 말고 Claude에게 요청

CLAUDE.md는 **Claude에게 업데이트 요청**하는 게 좋습니다.

```
CLAUDE.md에 아래 내용 추가해줘:
- 항상 한국어로 대화해줘
- 나는 마케터라서 코드보다 기획 관점으로 설명해줘
```

> 직접 수정하면 형식이 깨지거나 중복이 생길 수 있어요.
> Claude에게 요청하면 기존 내용과 자연스럽게 합쳐줍니다.

### 습관 2: 실수하면 지침으로 남기기

작업하다가 에러가 나거나 실수를 했다면?

```
왜 이런 에러가 났어?
다시는 같은 실수 반복하지 않도록 CLAUDE.md에 지침 추가해줘.
```

이렇게 하면 **Claude가 같은 실수를 반복하지 않습니다.**

### 습관 3: 자주 참조하는 파일 위치 적어두기

CLAUDE.md에 **"이 정보는 이 파일에 있어"**라고 적어두면 효율이 올라갑니다.

```markdown
## 주요 파일 위치
- 설계서: docs/설계서.md
- 설문 결과: data/survey-results.md
- 스킬 파일: .claude/skills/내스킬/SKILL.md
```

| 위치 안 적어두면 | 위치 적어두면 |
|----------------|-------------|
| Claude가 폴더 전체 뒤짐 | **바로** 해당 파일 열기 |
| 시간도 걸리고 토큰도 소모 | 즉시 작업 시작 |

> 💡 **비유**: 신입에게 "계약서가 어딨죠?"
> ❌ "공유드라이브 어딘가..." → 30분 헤맴
> ✅ "A팀 폴더 > 계약서 > 2026년요" → 바로 찾음

### 습관 4: 상황별 업데이트

| 상황 | 이렇게 하세요 |
|------|-------------|
| 새 프로젝트 시작 | `/init` 실행 → 기본 설정 완료 |
| 파일이 많이 바뀌었을 때 | `/init` 다시 실행 |
| Claude가 맥락을 잊는 것 같을 때 | "CLAUDE.md 업데이트해줘" |

---

## 핵심 정리

| 개념 | 설명 |
|------|------|
| CLAUDE.md | Claude의 뇌. 적어두면 매번 자동으로 기억 |
| `/init` | 폴더를 분석해서 CLAUDE.md를 자동 생성하는 명령어 |
| 저장 위치 | 프로젝트용 + 개인용 = 합쳐서 적용 |
| 업데이트 | 직접 수정 X → Claude에게 요청 |
| 핵심 습관 | 실수→지침 남기기, 파일 위치 적기, 상황별 /init |

> 🎯 **핵심**: CLAUDE.md는 한 번 만들고 끝이 아닙니다.
> 프로젝트하면서 **계속 업데이트**하는 살아있는 문서예요.
