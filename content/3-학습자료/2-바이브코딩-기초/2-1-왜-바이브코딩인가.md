## 💡 "코딩" 안 해도 바이브코딩입니다

**바이브코딩 ≠ 코드 짜기**

여러분이 하는 일이 꼭 "코딩"이 아니어도 됩니다.

| 직무 | 바이브코딩 예시 |
|------|----------------|
| **PM** | "이 PRD 문서 읽고 유저 스토리 정리해줘" |
| **디자이너** | "이 폴더의 피드백들 분석해서 패턴 찾아줘" |
| **리서처** | "인터뷰 녹취록 5개 읽고 인사이트 뽑아줘" |
| **마케터** | "경쟁사 분석 자료 만들어줘" |
| **누구나** | "회의록 정리해줘", "보고서 초안 써줘" |

**핵심은 "코드"가 아니라 "내 작업 환경에서 AI가 직접 일한다"는 것.**


## 🚀 여기에 "스킬"까지 더하면?

> Part 1에서 스킬 개념을 배웠죠? 이게 왜 게임 체인저인지 이제 이해됩니다.

> 💡 **실제 사례**: 강사가 `/billing` 명령어 하나로 96분 → 10분으로 줄인 이야기, Part 1에서 보셨죠? 이게 바이브코딩입니다.

**스킬 = 내가 정의한 업무 지침서**

반복하는 작업이 있다면? **스킬로 만들어두면 한 마디로 끝.**

```
❌ 매번: "인터뷰 녹취록 읽고, 핵심 발언 추출하고, 감정 분석하고, 
        인사이트 정리해서, 요약 문서 만들어줘. 형식은 이렇게..."

✅ 스킬: "/인터뷰분석"  ← 끝
```

**직무별 스킬 예시:**

| 직무 | 반복 업무 | 스킬로 만들면 |
|------|----------|--------------|
| **PM** | PRD → 유저스토리 변환 | `/prd-to-story` |
| **디자이너** | 피드백 정리 → 액션아이템 | `/피드백정리` |
| **리서처** | 인터뷰 → 인사이트 추출 | `/인터뷰분석` |
| **마케터** | 경쟁사 → 비교 분석표 | `/경쟁사분석` |
| **누구나** | 회의 → 회의록 + 할일 | `/회의록` |

**스킬의 힘:**

```
일반 AI 사용:
"회의록 정리해줘" → 매번 형식 설명 → 매번 다른 결과

스킬 사용:
"/회의록" → 항상 같은 형식 → 우리 팀 스타일로 → 자동 저장까지
```

| 일반 사용 | 스킬 사용 |
|----------|----------|
| 매번 프롬프트 작성 | **한 단어**로 실행 |
| 매번 형식 설명 | **일관된** 결과물 |
| 결과 품질 들쭉날쭉 | **검증된** 워크플로우 |
| 나만 사용 가능 | **팀과 공유** 가능 |

> 💡 **오늘 워크샵 목표**: 여러분만의 스킬을 만들어서 GitHub에 배포!
> 그러면 언제 어디서든 `/내스킬` 한 마디로 업무 자동화.
